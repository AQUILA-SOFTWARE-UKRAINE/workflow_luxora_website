## ✅ Виконано

### 1. Купівля `.de` домену через INWX
Домен `luxorareinigung.de` зареєстровано напряму (без Trustee Service — оскільки бізнес офіційно зареєстрований і власник фізично проживає в Німеччині, DENIC вимагає лише німецьку адресу, а не громадянство).

### 2. Налаштування DNS
В INWX додано записи, що вказують на сервер:
- `A` (`@`) → `167.233.235.181`
- `AAAA` (`@`) → `2a01:4f8:c015:cbb7::1`
- (аналогічно для `www`)

Перевірено через `nslookup luxorareinigung.de` — резолвиться коректно на обидві адреси.

### 3. Створення Hetzner Cloud сервера
Створено сервер `luxora-prod-01`:
- Тип: **CX23** (2 vCPU / 4GB RAM / 40GB NVMe SSD) — прямий наступник знятого з продажу CX22
- Образ: Ubuntu 24.04
- Локація: Nuremberg/Falkenstein (Німеччина)
- Доступ по SSH-ключу (без пароля) від самого початку

### 4. Хардненг сервера
- Створено sudo-користувача `deploy`, скопійовано SSH-ключ (`rsync --archive --chown`)
- Перевірено вхід під `deploy` **до** вимкнення root/пароля (щоб не заблокувати доступ)
- `PermitRootLogin no`, `PasswordAuthentication no` в `sshd_config`
- Встановлено та увімкнено `fail2ban` (захист SSH від брутфорсу)
- Увімкнено `unattended-upgrades` (автооновлення безпекових пакетів)
- Налаштовано **Hetzner Cloud Firewall**: дозволено лише 22/80/443, решта — заблоковано
- Перевірено `nmap`-скануванням ззовні: 22 `open`, 80/443 `closed` (не `filtered` — фаєрвол пропускає, просто сервіс ще не запущено), 3389 `filtered` (заблоковано) — все як і треба

### 5. Встановлення Docker
- Docker Engine + Compose plugin встановлено з офіційного репозиторію Docker для Ubuntu 24.04
- Користувач `deploy` доданий у групу `docker` (працює без `sudo`)
- `docker.service` увімкнено в автозапуск
- Перевірено: `docker --version`, `docker compose version`, `docker run hello-world` — все відпрацювало успішно
